## Project

Photonic is a self-hosted photo management system targeting homelab/NAS deployments. It
consists of a Rust backend that exposes a REST API and a native iOS/macOS client that
consumes it.

## API Contract (server в†” apple)

`openapi.yaml` at the repo root is the **single source of truth** for the HTTP API. It is
generated by the server and consumed by both the server's own integration-test client and the
Swift app.

- The server generates it from utoipa annotations via `cd server && cargo xtask generate-openapi`
- The Swift app reads it via a symlink: `apple/Photonic/openapi.yaml в†’ ../../openapi.yaml`
- Both clients regenerate their typed bindings from this one file

### Regenerating the API client after a server-side change

1. Update Rust handlers + utoipa annotations under `server/crates/`
2. `cd server && cargo xtask generate-openapi` вЂ” rewrites root `openapi.yaml`
3. On the server: `cargo build` auto-regenerates the Rust progenitor client
4. In Xcode: rebuild вЂ” `swift-openapi-generator` picks up the updated spec via the symlink

Similarly, `asyncapi.yaml` at the repo root defines the event bus contract. It is consumed by
the server's `build.rs` to generate Rust event DTOs.

## Entry Points

- **Server dev loop**: `nix develop .#test` (from repo root) вЂ” sets up a local Postgres in
  `server/tmpdata/`, runs migrations, drops you into a shell where `cargo test` works.
- **Swift app dev loop**: open `apple/Photonic.xcodeproj` in Xcode.
- **Local server + deps**: `docker-compose up` at the repo root.
